Portability

Platforms and compilers

The table below summarizes the platforms and compilers which Duktape is known to work on, with portability notes where appropriate. This is not a list of supported platforms, rather a list of what is known to work.

Operating system Compiler Processor Notes
Linux GCC x86, x86-64, ARM, MIPS
AmigaOS VBCC M68K Must define AMIGA and __MC68K__ manually (no platform or processor defines available).
TOS (Atari ST) VBCC M68K Must define __TOS__ and __MC68K__ manually (no platform or processor defines available).

Using C++ instead of C

There are two ways to use Duktape in a C++ program:

  • Compile Duktape in plain C, and use it from a C++ program. The duktape.h header provides the necessary glue to avoid name mangling issues.
  • Compile both Duktape and your program as C++..

Limitations

  • The int type is assumed to be 32 bits. This is incorrect even on some platforms which provide a 32-bit type.
